Motivation

Galaxy's domain-agnostic platform extends well beyond bioinformatics. The Digital Humanities and Social Sciences SIG connects researchers who need computational tools for analyzing large text corpora but often lack programming expertise.

We integrated two leading NLP frameworks into Galaxy, enabling researchers to perform sophisticated text analysis without writing a single line of code.

Stanford CoreNLP

Java-based NLP toolkit from Stanford University providing a unified annotation pipeline.

Annotators:

  • Tokenization & sentence splitting
  • Part-of-speech tagging & lemmatization
  • Named entity recognition (NER)
  • Dependency & constituency parsing
  • Coreference resolution
  • Sentiment analysis

Languages:

English Chinese Arabic French German Spanish Italian Hungarian

spaCy

Python-based NLP library designed for production use with fast, accurate processing.

Annotators:

  • Tokenization & sentence segmentation
  • Part-of-speech tagging & lemmatization
  • Named entity recognition (NER)
  • Dependency parsing

25+ languages with models in multiple sizes:

sm — fast, compact md — balanced lg — accurate trf — transformer

Architecture

Each tool follows a consistent pattern designed for portability and ease of administration.

Galaxy Tool Wrapper
↓
Docker Container
Data Manager
↓
Language Models
Data Tables
  • Tool wrappers define inputs, outputs, and the command pipeline in Galaxy XML
  • Docker containers provide isolated, reproducible execution environments
  • Data managers download language models on demand and register them in Galaxy's data tables
  • Data tables present installed models as selectable dropdowns in the tool interface

Output Formats

Both tools support multiple standard NLP output formats for interoperability:

JSON CoNLL CoNLL-U Plain Text XML (CoreNLP)

Structured outputs enable downstream processing with other Galaxy tools or external analysis pipelines.

For Administrators

  • Install tools from the Galaxy Toolshed
  • Use data managers (Admin → Local Data) to install only the language models your researchers need
  • No manual configuration of model files or paths required
  • CoreNLP: select languages from 8 options
  • spaCy: choose from 70+ models across 25+ languages in multiple sizes

Why Not Just Use an LLM?

  • Reproducibility — CoreNLP and spaCy produce identical results on every run. LLMs are non-deterministic: the same prompt can yield different entity tags or miss entities across runs. For citable, replicable research this matters.
  • Structured output — Standard NLP formats (CoNLL-U, JSON with token offsets, dependency trees) are machine-parseable and consistent. LLM output is free text requiring additional parsing with no schema guarantees.
  • Scale and cost — Processing a 120-chapter novel or a 500-document corpus costs nothing beyond compute time. LLM APIs charge per token and are rate-limited, making large-scale text analysis expensive.
